Semantic HTML

Meaningful Markup for Accessibility

Why is it so important and who is it for?

Any consumer of HTML, our B.U.D.S.S.

  • Browsers
  • Users
  • Developers
  • Screen Readers
  • Spiders/Scrapers

Browser Benefits

Better keyboard access, default focus handling for sementic elements, and better browser native interactions.

User Benefits

Can consume markup with the browser or any assistive device, can disable stylesheets or use custom stylesheets to enhance semantic markup.

Developers

Can derive meaning from the markup used as to the intent of a template or component. Ex:


export default Component.extend({
    tagName: 'button',
    classNameBindings: [
        'someClass::nope',
        'disabled:disabled',
    ]
});

Just from observing the `tagName` you get an idea of the funtionaly inteneded.

Screen Readers

Can now help users understand the meaning of content being read and unlock additional navigational options based on the tags used

More on this per element...

Spiders/Scrapers

Appropriate roles and markup, such as navigation, let robots scrape your content much easier. Google can automatically generate a sitemap just from markup. Semantic markup can allow scapers to get your content without configuration.

Win Win Win

All consumers of semantic HTML benefit.

Good Deal... Maybe

  • Headers call out the start of sections of content, hence the full name "Section Heading"
  • Screen readers can use headings to skip through large sections of content.
  • H1's are used as the title of a page, it's not reccomended to have multiple H1's. Spider/Scraper benefit from this as well.
  • Should be part of base typography, shouldn't need much styling.

Don't Overuse

You shouldn't use headings for purely decorative purposes, instead use more unique callout styles. Which is why I said maybe before.



Paragraph title

If you're adding tons of styles to a heading so it fits in your widget, likely it shouldn't be a heading.

ARIA

MDN ARIA

ARIA provides a way to add the missing semantics needed by assistive technologies such as screen readers. ARIA enables developers to describe their widgets in more detail by adding special attributes to the markup.

ARIA Roles

Roles describe widgets that aren't otherwise available in HTML 4, such as sliders, menu bars, tabs, and dialogs.
